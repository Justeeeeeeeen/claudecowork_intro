Claude Cowork is a desktop AI agent that works directly with your local files — documents, spreadsheets, presentations — from inside the Claude Desktop app. Unlike Claude Chat or Claude Code, Cowork can open, read, edit, and create files on your machine and run tasks autonomously. No coding skills needed.

What You Need to Install
🖥️
Claude Desktop App

The main app that runs Cowork. Download, install, sign in, then click the Cowork tab in the sidebar. On first run it downloads a small VM image — just let it complete.

↓ claude.ai/download
🐍
Python

Cowork uses Python internally to run automations. You don't write any code — it just needs to be installed on your machine.

↓ python.org/downloads Version 3.11 or newer
⚠️ During installation, tick "Add Python to PATH" — easy to miss
⬡
Node.js

Required by some Cowork plugins and connectors. Not needed immediately but good to have for when you want to extend Cowork's capabilities.

↓ nodejs.org Download the LTS version
🔌
MCP Connectors

How Cowork connects to Gmail, Google Calendar, Google Drive, and other services. Once linked, Cowork can read emails, check your calendar, and pull data — all without switching tabs.

⚙️ Claude Desktop → Settings → Connectors

Why Cowork?
The shift from answering questions to actually doing work

You might already use Claude Chat (claude.ai) to draft emails or answer questions, or Claude Code to help write scripts. Both are reactive — you ask, they respond, and the job is done when the conversation ends.

Cowork is different. It's an agent that lives on your machine. It can open your real files, connect to your tools, run multi-step tasks, and operate on a schedule — all without you prompting every step.

Three Reasons It Matters for SPX Work
⏱️
Eliminates repetitive manual work
Anything you do on a schedule — formatting reports, copying data between files, drafting the same type of email — Cowork can do for you automatically.
SPX example: Instead of manually compiling your weekly 3WH utilisation report every Monday, Cowork reads the CSV, formats it, pulls insights, and drafts the email — you just review and send.
📁
Works with your actual files
Unlike browser Claude where you paste content in, Cowork is given access to your real folders. It can read a spreadsheet, edit a Word doc, and save a new file — all in one go.
SPX example: "Read all the dispute logs from this week's folder, find patterns, and write a summary" — Cowork does it across every file, not just one you paste in.
🤖
Agents that think, chain, and interact
This is the big idea. An agent isn't just Claude answering one question — it's Claude reasoning through a multi-step task, using tools (Gmail, Calendar, your files), making decisions, and looping back when something needs checking.
Think of it like: A junior analyst who has access to your laptop. You leave them a note about what needs doing, and when you come back, it's done — with decisions logged so you can review them.

01
Markdown is just a file type — like PDF or DOCX.
Claude reads it best.
File types you already know
PDF
Fixed layout. Hard for AI to parse — text is trapped in visual formatting.
DOCX
XML under the hood. Lots of hidden tags that get in the way.
.md
Pure text. No hidden formatting. Claude reads every character exactly as written.
Why .md is Claude's native language
No binary encoding — it's plain text, so Claude reads it as-is with no translation layer.
Unambiguous structure — # means heading, - means bullet. No guessing.
No noise — unlike DOCX or PDF, there are no hidden XML tags or layout instructions cluttering the content.
# SPX Weekly Report - 3WH utilisation: 87% - Active vehicles: 142 **Action needed:** Review hub B
Open any .md file in Notepad — it's just text.
02
CLAUDE.md is the briefing note Cowork reads before every session.
It tells Claude who you are and how to behave.
Without CLAUDE.md, every session starts blank. With it, Claude already knows your context before you type a word.
What goes in CLAUDE.md
# SPX Philippines — Context We are a last-mile logistics company. Vehicle types: 2WH, 3WH, 4WH. Language: English only. Never send emails without asking me first. Always put a summary at the top of reports.
Who we are Rules & preferences Output format
Two-layer structure
📁 work/ ├── CLAUDE.md ← always loaded │ "We are SPX PH, 3 vehicle types…" ├── 📁 3wh-email/ │ ├── CLAUDE.md ← task instructions │ └── fleet_mix_raw.csv └── 📁 3wh-disputes/ ├── CLAUDE.md ← escalation rules └── disputes_log.xlsx
Root = who we are. Task folder = what to do. Claude reads both, in that order.

PART B
2
Use Claude to create your CLAUDE.md
CLAUDE.md is for things that are always true about the task — the format, tone, recipients, and rules that apply every single time. We'll do this in 3 mini-steps: generate a draft → update your details → ask Claude to refine it.
What a good CLAUDE.md looks like — copy this as your template
1 — REPORT PURPOSE
## Report Purpose
[What this report is, how often it goes out, and what
decision or action it supports]
e.g. Weekly fleet utilisation summary sent every Monday.
    Flags hubs below the 85% 3WH threshold.
2 — DATA SOURCES
## Data Sources
- Primary: `[your_file.csv]` — uploaded each time
- Key columns: [list what matters]
- Period covered: [daily / weekly / monthly]
e.g. fleet_mix_raw.csv — hub_name, vehicle_type, utilisation_pct
3 — EMAIL STRUCTURE
## Email Structure
1. Subject line — [what it should include]
2. Opening summary — [1–3 sentence overview]
3. Key metrics — [table, bullets, or cards]
4. Highlights / flags — [what to surface]
5. Closing — [action items or next steps]
4 — RECIPIENTS
## Recipients
- To: [name / role / email]
- CC: [optional]
- Audience: [what they care about, how they read
  data — e.g. data-literate, prefers concise summaries]
5 — ALWAYS DO
## Always
- [Your quality rules]
- [What to flag or check]
e.g. Flag if any hub data is missing
6 — NEVER DO
## Never
- Guess or fill in missing data
- Change the recipient list
- [Any rules specific to your team]
STEP 2A — GENERATE DRAFT
Ask Claude to create a first draft
"Using the CLAUDE.md template structure above, create a first draft CLAUDE.md file in my current folder for my [describe your email — e.g. weekly fleet mix report / daily ops summary / hub dispute report]. Use sensible defaults — I'll review and update the details next."
STEP 2B — PERSONALISE
Open the CLAUDE.md file Claude just created and update these fields with your real details:
MUST UPDATE
Recipients — replace with your actual team email and manager's email
MUST UPDATE
Report Purpose — describe what decisions your email actually supports
OPTIONAL
Tone & Style — adjust if your team has specific preferences
OPTIONAL
Never section — add anything specific to your team's rules
STEP 2C — REFINE WITH CLAUDE
Ask Claude to refine based on your updates
"I've updated the CLAUDE.md with my real recipient list and report purpose. Re-read the file and refine it — make sure the tone, rules, and email structure are consistent with the recipients and purpose I've described. Save the updated version."
💡 This draft → personalise → refine loop is how you should set up any new task in Cowork. Claude does the heavy lifting; you just correct the details it couldn't know.
PART B
Build your Knowledge Base
Why a KB? The KB gives Cowork the detail it needs to write accurate, context-aware emails — vehicle specs, past email examples, or your SOP for what "good utilisation" means.
🎨 Where does email style live?
CLAUDE.md
Style rules in plain English
Things always true about the format — colours, row logic, section order, tone. Write it once, applies every week automatically.
e.g. "Use orange header, dark navy insight panel, colour rows red/amber/green"
Knowledge Base
A real email to copy
Drop a past email into the kb/ folder as a reference. Claude looks it up when generating to match the exact structure.
e.g. email_style_guide.md extracted from your best email
Skill
Just the trigger
The skill calls both CLAUDE.md and the KB — but it doesn't store the style itself. Think of it as the "run" button.
e.g. "Generate fleet mix email" → triggers everything
3
Create a kb/ folder and add your documents
Inside your task folder, create a kb/ subfolder. Add any documents that help Cowork understand the task — targets, definitions, SOPs, or a past email to reference.
📁 my-weekly-email/ ├── CLAUDE.md ├── your_data_file.csv └── 📁 kb/ ├── targets_or_thresholds.xlsx ← what good looks like ├── definitions_or_sop.pdf ← context Claude needs to know └── email_style_guide.md ← layout, tone, format rules
💡 Style tip: Instead of saving a raw email file, write a email_style_guide.md — plain English describing your layout, colours, and tone. Claude reads it easily and you can share it with teammates so everyone's email looks the same.
Using the 3WH example? Download fleet_mix_raw.csv + 3wh_utilisation_targets.xlsx from the sample files folder ↗ and drop them into your task folder and kb/.
4
Test that Cowork can use the KB
Before building the skill, verify Cowork can find and use your documents by asking a question it would need to search the KB to answer.
Try this prompt "Based on the documents in my kb/ folder, what is the target 3WH utilisation rate and how does it compare to what's in fleet_mix_raw.csv this week?"
If the answer seems off, check that your kb/ folder is inside the folder Cowork has access to.
⚠️ Before Part C — connect your Gmail
The skill saves drafts to Gmail. For this to work, Cowork needs permission to access your email. If you haven't done this yet, connect it now before building the skill.
How to connect:
Cowork sidebar → Connectors → Gmail → Connect → sign in with your Shopee Google account
You only need to do this once — Cowork will remember it for all future sessions.
PART C
Create the Skill
Why a skill? Instead of typing a long prompt every Monday, you'll have a saved skill you can trigger in seconds — and share with your team so everyone uses it the same way.
5
Ask Cowork to create the skill
You don't need to write the skill file yourself. Just describe what you want the skill to do and Cowork will create the file for you.
Try this prompt "Create a skill called '[your-email-name]'. When triggered, it should: read [your_data_file.csv], check the kb/ folder for any reference documents, generate the email using the format and rules in my CLAUDE.md, and save a draft in Gmail ready for me to review and send."
3WH example: "Create a skill called 'fleet-mix-email'. When triggered, read fleet_mix_raw.csv, check kb/ for utilisation targets, generate the weekly %3WH email using my CLAUDE.md format, and save a Gmail draft."
6
Review the skill file Cowork created
Cowork will save the skill as a .md file in your skills folder. Open it and read through it — make sure the instructions match what you want. Edit any part you want to change.
💡 The skill file is just a markdown file. You can open it in Notepad and edit it like any other text file.
PART D
Run it
7
Trigger your skill
Now test everything you've built. Open Cowork with your task folder selected and trigger the skill.
Try this prompt "Generate [your email name]"
3WH example: "Generate fleet mix email"
Cowork will read your data file, check your KB for context, and draft the email in the format you defined — using the rules from your CLAUDE.md throughout.
8
Review, refine, and confirm
Read the draft email Cowork produced. If anything is off, tell it what to change in plain English. Once it's right, ask Cowork to save it as a Gmail draft.
Example refinement prompts
"Make the subject line shorter"
"Add a line flagging that Hub B is below target"
"Save this as a Gmail draft to ops-team@spx.com"
9
Ask Claude to remember the changes
After any refinement, always close the loop — ask Claude to update your CLAUDE.md so the same fix applies automatically next week. This is how your automation gets better over time without extra effort.
Use this after every meaningful edit
"I've made some changes to this email. Please update my CLAUDE.md to reflect what I changed — so the next time you generate this email, these preferences are already applied."
What gets saved — examples
You said:
"Always bold the WoW delta column"
Saved to CLAUDE.md:
Always bold the week-on-week delta column in the metrics table.
You said:
"Don't include hubs with less than 10 deliveries"
Saved to CLAUDE.md:
Exclude hubs with fewer than 10 deliveries from the report.
💡 Think of this as training your assistant. Each time you catch something and save it, you're building a smarter starting point for next week — and every week after.
💡 Tips for getting better results
🧠 Your CLAUDE.md is the multiplier. The more context you put in it — abbreviations, what your team cares about, what "good" looks like — the better every output will be, not just this email.
📄 Add a style guide to the KB. Drop a past email (or write a plain-text description of your format) into kb/ as a reference. Cowork will match the tone, structure, and level of detail automatically — and you can share it with teammates.
⚡ Every edit is a lesson. When you catch something wrong — wrong tone, missing section, wrong number format — don't just fix it once. Tell Claude to save it to CLAUDE.md so it never happens again.
🔄 Skills are shareable. Once your email skill is working well, share the skill file with your team. They can install it and generate the same email from their own data.

What actually goes into a webpage?
Before we build, here's what every webpage is made of — and what Claude collapses for you.
🏗️
HTML
The skeleton. Defines what's on the page — headings, tables, buttons, cards. Like drawing boxes on a blueprint.
<table> <div> <h1>
🎨
CSS
The styling. Controls colours, fonts, spacing, layout. Like painting and decorating the blueprint.
color: #EE4D2D; font-size: 14px
⚙️
JavaScript (JS)
The logic. Sorts tables, filters data, calculates totals, shows/hides tabs. Everything interactive and dynamic.
sort(), filter(), calculate()
💡
Claude puts HTML + CSS + JS into a single .html file
You don't need to manage three separate files. Everything lives in one document — open it in any browser, share it by email, no installation required. This is the "magic" of what Claude builds.
